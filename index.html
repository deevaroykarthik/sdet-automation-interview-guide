1 Reverse a String Without Built-in Methods
JavaDeloitteAmazonEasy0-2 YOE
Time: O(n) | Space: O(n)
public static String reverse(String s) {
    char[] chars = s.toCharArray();
    int left = 0, right = chars.length - 1;
    while (left < right) {
        char temp = chars[left];
        chars[left++] = chars[right];
        chars[right--] = temp;
    }
    return new String(chars);
}

public static List<Integer> findDuplicates(int[] arr) {
    Set<Integer> seen = new HashSet<>();
    Set<Integer> duplicates = new LinkedHashSet<>();
    for (int num : arr) {
        if (!seen.add(num)) duplicates.add(num);
    }
    return new ArrayList<>(duplicates);
}
6 Two Sum (HashMap Approach)

Time: O(n) | Space: O(n)
public static int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement))
            return new int[]{map.get(complement), i};
        map.put(nums[i], i);
    }
    return new int[]{};
}

9 Balanced Parentheses (Stack)
JavaAmazonMicrosoftGoogleEasy0-3 YOE
Time: O(n) | Space: O(n)
public static boolean isValid(String s) {
    Stack<Character> stack = new Stack<>();
    for (char c : s.toCharArray()) {
        if (c == '(') stack.push(')');
        else if (c == '{') stack.push('}');
        else if (c == '[') stack.push(']');
        else if (stack.isEmpty() || stack.pop() != c)
            return false;
    }
    return stack.isEmpty();
}

11 Linked List Palindrome Check (No Extra Space)
JavaAmazon SDET-1Medium2-5 YOE
Time: O(n) | Space: O(1)
public boolean isPalindrome(ListNode head) {
    // 1. Find middle using slow/fast pointers
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    // 2. Reverse second half
    ListNode prev = null;
    while (slow != null) {
        ListNode next = slow.next;
        slow.next = prev;
        prev = slow;
        slow = next;
    }
    // 3. Compare first half with reversed second half
    ListNode left = head, right = prev;
    while (right != null) {
        if (left.val != right.val) return false;
        left = left.next;
        right = right.next;
    }
    return true;
}
Key: Find middle → reverse second half in-place → compare. No extra list/array. This exact problem was asked in Amazon SDET-1 Chennai interview.

13 Binary Tree Level Order Traversal (BFS)
JavaAmazonMicrosoftGoogleMedium2-5 YOE
Time: O(n) | Space: O(n)
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        int size = queue.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}
14 Rotate 2D Matrix by 90° (In-Place)
JavaAmazon SDET-1MicrosoftMetaMedium3-5 YOE
Time: O(n²) | Space: O(1)
public void rotate(int[][] matrix) {
    int n = matrix.length;
    // Step 1: Transpose (swap rows with columns)
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++) {
            int temp = matrix[i][j];
            matrix[i][j] = matrix[j][i];
            matrix[j][i] = temp;
        }
    // Step 2: Reverse each row
    for (int i = 0; i < n; i++) {
        int l = 0, r = n - 1;
        while (l < r) {
            int temp = matrix[i][l];
            matrix[i][l++] = matrix[i][r];
            matrix[i][r--] = temp;
        }
    }
}
Trick: Transpose + Reverse each row = 90° clockwise rotation. This was asked in Amazon SDET-1 coding round.

16 Longest Substring Without Repeating Characters
JavaAmazonGoogleMetaMicrosoftMedium3-6 YOE
Time: O(n) | Space: O(min(n, 128))
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> map = new HashMap<>();
    int maxLen = 0, left = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (map.containsKey(c))
            left = Math.max(left, map.get(c) + 1);
        map.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
Sliding Window + HashMap. Classic FAANG SDET question. Maintain a window with no duplicates — shrink from left when duplicate found.
17 Maximum Subarray Sum (Kadane's Algorithm)
JavaMicrosoftAmazonMedium2-5 YOE
Time: O(n) | Space: O(1)
public int maxSubArray(int[] nums) {
    int maxSum = nums[0], curSum = nums[0];
    for (int i = 1; i < nums.length; i++) {
        curSum = Math.max(nums[i], curSum + nums[i]);
        maxSum = Math.max(maxSum, curSum);
    }
    return maxSum;
}
Kadane's: At each index, either start a new subarray or extend the current one. Microsoft Round 2 frequently asks this.

public char firstUniqChar(String s) {

19 Merge Two Sorted Linked Lists
JavaAmazonMicrosoftEasy1-3 YOE 
public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            curr.next = l1; l1 = l1.next;
        } else {
            curr.next = l2; l2 = l2.next;
        }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
20 3Sum — Find All Triplets Summing to Zero
JavaGoogleMetaAmazonMedium3-6 YOE
Time: O(n²) | Space: O(1) excl output
public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);
    List<List<Integer>> res = new ArrayList<>();
    for (int i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) continue; // skip dupes
        int lo = i + 1, hi = nums.length - 1;
        while (lo < hi) {
            int sum = nums[i] + nums[lo] + nums[hi];
            if (sum == 0) {
                res.add(Arrays.asList(nums[i], nums[lo], nums[hi]));
                while (lo < hi && nums[lo] == nums[lo+1]) lo++;
                while (lo < hi && nums[hi] == nums[hi-1]) hi--;
                lo++; hi--;
            } else if (sum < 0) lo++;
            else hi--;
        }
    }
    return res;
}
Sort + Two Pointers. Fix one element, then use two pointers on the remaining sorted array. Skip duplicates carefully. Top FAANG SDET question.
21 LRU Cache Implementation
JavaAmazonGoogleMicrosoftMedium5+ YOE
Time: O(1) get/put | Space: O(capacity)
class LRUCache extends LinkedHashMap<Integer, Integer> {
    private int capacity;

    public LRUCache(int capacity) {
        super(capacity, 0.75f, true); // accessOrder=true
        this.capacity = capacity;
    }

    public int get(int key) {
        return super.getOrDefault(key, -1);
    }

    public void put(int key, int value) {
        super.put(key, value);
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry e) {
        return size() > capacity;
    }
}
Amazon SDET VO variant. LinkedHashMap with accessOrder=true handles LRU ordering. Override removeEldestEntry for eviction. For interviews, also know the manual DoublyLinkedList + HashMap approach.
22 Number of Islands (Graph/DFS)
JavaAmazonGoogleMetaMedium3-6 YOE
Time: O(m×n) | Space: O(m×n)
public int numIslands(char[][] grid) {
    int count = 0;
    for (int i = 0; i < grid.length; i++)
        for (int j = 0; j < grid[0].length; j++)
            if (grid[i][j] == '1') { count++; dfs(grid, i, j); }
    return count;
}
void dfs(char[][] g, int i, int j) {
    if (i<0||i>=g.length||j<0||j>=g[0].length||g[i][j]=='0') return;
    g[i][j] = '0';
    dfs(g,i+1,j); dfs(g,i-1,j); dfs(g,i,j+1); dfs(g,i,j-1);
}
Classic FAANG graph problem. Each DFS flood-fills one island. Count how many times DFS is triggered from a '1' cell.
23 Spiral Matrix Traversal
JavaMicrosoftAmazonAppleMedium3-5 YOE
Time: O(m×n) | Space: O(1) excl output
public List<Integer> spiralOrder(int[][] matrix) {
    List<Integer> res = new ArrayList<>();
    int top = 0, bottom = matrix.length - 1;
    int left = 0, right = matrix[0].length - 1;
    while (top <= bottom && left <= right) {
        for (int j = left; j <= right; j++) res.add(matrix[top][j]);
        top++;
        for (int i = top; i <= bottom; i++) res.add(matrix[i][right]);
        right--;
        if (top <= bottom) {
            for (int j = right; j >= left; j--) res.add(matrix[bottom][j]);
            bottom--;
        }
        if (left <= right) {
            for (int i = bottom; i >= top; i--) res.add(matrix[i][left]);
            left++;
        }
    }
    return res;
}
Layer-by-layer traversal. Maintain 4 boundaries (top, bottom, left, right) and shrink after each direction. Microsoft and Apple SDET favorite.
24 Producer-Consumer Pattern (Multithreading)
JavaAmazonMicrosoftHard5+ YOE 
class BoundedBuffer<T> {
    private final Queue<T> queue = new LinkedList<>();
    private final int capacity;

    public BoundedBuffer(int capacity) { this.capacity = capacity; }

    public synchronized void produce(T item)
            throws InterruptedException {
        while (queue.size() == capacity) wait();
        queue.add(item);
        notifyAll();
    }

    public synchronized T consume()
            throws InterruptedException {
        while (queue.isEmpty()) wait();
        T item = queue.poll();
        notifyAll();
        return item;
    }
}
Senior SDET must-know. Demonstrates thread synchronization, wait/notify, and bounded buffers. Critical for understanding parallel test execution internals. Amazon and Microsoft ask this for senior SDET roles.
25 Serialize & Deserialize Binary Tree
JavaGoogleAmazonHard5+ YOE
Time: O(n) | Space: O(n)
public String serialize(TreeNode root) {
    if (root == null) return "null";
    return root.val + "," + serialize(root.left)
                     + "," + serialize(root.right);
}

public TreeNode deserialize(String data) {
    Queue<String> q = new LinkedList<>(
        Arrays.asList(data.split(",")));
    return build(q);
}

private TreeNode build(Queue<String> q) {
    String val = q.poll();
    if (val.equals("null")) return null;
    TreeNode node = new TreeNode(Integer.parseInt(val));
    node.left = build(q);
    node.right = build(q);
    return node;
}
Google/Amazon bar-raiser level. Pre-order traversal with null markers. Demonstrates recursion, string manipulation, and tree construction — all in one problem.

SECTION 2

Framework Design Interview Questions & Answers

The most critical section — every SDET interview asks "Explain your automation framework." Here's how to answer.

Q1. What is Page Object Model (POM) and why do you use it?
POM is a design pattern where each web page is represented as a Java/JS class containing the page's element locators and action methods. Tests call page methods instead of directly interacting with elements.

Why: If a UI element changes, you update one page class — not 50 test files. It gives you separation of concerns (locators vs test logic), code reusability, and readability. Example: loginPage.enterUsername("admin") is clearer than driver.findElement(By.id("user")).sendKeys("admin").
Q2. What design patterns do you use in your framework?
1. Page Object Model — separates UI locators from test logic
2. Singleton Pattern — ensures one WebDriver instance per thread (via ThreadLocal for parallel execution)
3. Factory Pattern — creates browser instances (Chrome, Firefox, Edge) based on config without exposing creation logic
4. Fluent Page Object — method chaining: loginPage.enterUser("x").enterPass("y").clickLogin()
5. Data-Driven Pattern — separates test data from test logic using Excel/JSON/CSV/DataProviders
Q3. Explain your framework architecture end-to-end.
A well-designed framework has these layers:

1. Config Layer — property/YAML files for URLs, browser type, timeouts, environment
2. Driver Management Layer — WebDriver factory with ThreadLocal for parallel safety
3. Page Object Layer — one class per page with locators + action methods
4. Test Layer — test classes using TestNG/JUnit/Playwright Test annotations
5. Utilities Layer — helpers for waits, screenshots, logging, data readers
6. Reporting Layer — Allure/ExtentReports/Playwright HTML reports
7. CI/CD Layer — Jenkins/GitHub Actions pipeline running tests on every commit
Q4. How do you handle dynamic elements / flaky tests?
Dynamic Elements: Use robust locators — relative XPath, CSS with attributes, data-testid. Avoid absolute XPath. Use explicit waits (WebDriverWait / auto-wait in Playwright).

Flaky Tests: Identify root cause first — is it timing? data dependency? environment? Then: use explicit waits instead of Thread.sleep(), ensure test data isolation, use retry mechanisms (TestNG IRetryAnalyzer / Playwright retries config), and run tests in isolated browser contexts.
Q5. Selenium vs Playwright — when would you choose which?
Choose Selenium when: Team knows Java, existing Selenium codebase, need to support older browsers, large community support needed.

Choose Playwright when: Starting fresh, need built-in auto-waiting (no explicit waits), need API + UI testing in one framework, need multi-browser with single API, need faster execution via DevTools protocol (no WebDriver layer), need built-in tracing/video/screenshots.

Key difference: Selenium communicates via WebDriver protocol (HTTP). Playwright uses WebSocket via DevTools protocol — faster and more reliable. Playwright also provides browser context isolation out of the box.
Q6. What is the Testing Pyramid?
Unit Tests (base, most) → fast, isolated, test individual functions
Integration/API Tests (middle) → test service interactions, API contracts
E2E/UI Tests (top, fewest) → test full user workflows, slowest and most brittle

As an SDET, you should advocate for more API tests and fewer flaky UI tests. UI tests should cover critical user journeys only. This gives faster feedback and lower maintenance cost.
Q7. How do you handle test data management?
Strategies:
• External files — JSON/Excel/CSV for data-driven tests
• Builder pattern — create test data objects programmatically
• API seeding — create test data via API before UI tests
• Database utilities — query/reset DB state before/after tests
• Faker libraries — generate random realistic data to avoid conflicts

Key principle: Each test should create its own data and clean up after itself. Never depend on data from another test.
Q8. How do you integrate tests with CI/CD?
Pipeline stages:
1. Code commit triggers pipeline (Jenkins/GitHub Actions/GitLab CI)
2. Build project (Maven/npm install)
3. Run unit tests → fast feedback gate
4. Run API tests → validate backend contracts
5. Run UI smoke tests → critical path validation
6. Generate reports (Allure/HTML) and publish artifacts
7. Notify team via Slack/email on failure

Best practice: Run full regression nightly, smoke suite on every PR. Use Docker for consistent environments. Use Selenium Grid or Playwright's built-in parallelism for speed.

Step 2: DriverFactory (Singleton + ThreadLocal)

public class DriverFactory {

    // ThreadLocal ensures each thread gets its own WebDriver
    private static ThreadLocal<WebDriver> tlDriver = new ThreadLocal<>();

    public static WebDriver initDriver(String browser) {
        switch (browser.toLowerCase()) {
            case "chrome":
                WebDriverManager.chromedriver().setup();
                tlDriver.set(new ChromeDriver());
                break;
            case "firefox":
                WebDriverManager.firefoxdriver().setup();
                tlDriver.set(new FirefoxDriver());
                break;
            case "edge":
                WebDriverManager.edgedriver().setup();
                tlDriver.set(new EdgeDriver());
                break;
        }
        getDriver().manage().window().maximize();
        getDriver().manage().timeouts()
            .implicitlyWait(Duration.ofSeconds(10));
        return getDriver();
    }

    public static WebDriver getDriver() {
        return tlDriver.get();
    }

    public static void quitDriver() {
        if (tlDriver.get() != null) {
            tlDriver.get().quit();
            tlDriver.remove();  // prevent memory leaks
        }
    }
}

Step 3: BasePage (Common Actions)

public class BasePage {
    protected WebDriver driver;
    protected WebDriverWait wait;

    public BasePage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(15));
    }

    protected void click(By locator) {
        wait.until(ExpectedConditions.elementToBeClickable(locator)).click();
    }

    protected void type(By locator, String text) {
        WebElement el = wait.until(
            ExpectedConditions.visibilityOfElementLocated(locator));
        el.clear();
        el.sendKeys(text);
    }

    protected String getText(By locator) {
        return wait.until(
            ExpectedConditions.visibilityOfElementLocated(locator)).getText();
    }
}

Step 3: BasePage

import { Page, Locator } from '@playwright/test';

export class BasePage {
  readonly page: Page;

  constructor(page: Page) {
    this.page = page;
  }

  async navigate(path: string) {
    await this.page.goto(path);
  }

  async click(locator: Locator) {
    await locator.click();  // auto-waits!
  }

  async fill(locator: Locator, text: string) {
    await locator.fill(text);
  }

  async getText(locator: Locator): Promise<string> {
    return await locator.textContent() ?? '';
  }
}

Step 4: LoginPage

import { Page } from '@playwright/test';
import { BasePage } from './BasePage';

export class LoginPage extends BasePage {
  private usernameInput = this.page.locator('#username');
  private passwordInput = this.page.locator('#password');
  private loginBtn      = this.page.locator('#login-btn');
  private errorMsg      = this.page.locator('.error');

  constructor(page: Page) { super(page); }

  async login(user: string, pass: string) {
    await this.fill(this.usernameInput, user);
    await this.fill(this.passwordInput, pass);
    await this.click(this.loginBtn);
  }

  async getErrorMessage() {
    return await this.getText(this.errorMsg);
  }
}

import { test, expect } from '@playwright/test';

test.describe('Login Tests', () => {

  test('valid login redirects to home', async ({ page }) => {
    const loginPage = new LoginPage(page);
    await loginPage.navigate('/login');
    await loginPage.login('admin', 'pass123');
    await expect(page).toHaveURL('/home');
    await expect(page.locator('.welcome'))
      .toHaveText('Welcome, Admin');
  });

  test('invalid login shows error', async ({ page }) => {
    const loginPage = new LoginPage(page);
    await loginPage.navigate('/login');
    await loginPage.login('wrong', 'wrong');
    await expect(page.locator('.error'))
      .toContainText('Invalid credentials');
  });
});

Step 3: Step Definitions

public class UserAPISteps {
    private Response response;
    private RequestSpecification request;

    @Given("the API base URL is configured")
    public void setBaseUrl() {
        request = RestAssured.given()
            .baseUri(ConfigReader.get("baseUrl"))
            .contentType(ContentType.JSON)
            .log().all();
    }

    @When("I send a POST request to {string} with:")
    public void sendPost(String endpoint, DataTable table) {
        Map<String, String> data = table.asMap(String.class, String.class);
        response = request
            .body(data)
            .post(endpoint);
    }

    @When("I send a GET request to {string}")
    public void sendGet(String endpoint) {
        response = request.get(endpoint);
    }

    @Then("the response status code should be {int}")
    public void verifyStatusCode(int code) {
        Assert.assertEquals(response.getStatusCode(), code);
    }

    @Then("the response body should contain {string}")
    public void verifyBodyContains(String text) {
        Assert.assertTrue(response.getBody().asString().contains(text));
    }

    @Then("the response should have field {string} with value {int}")
    public void verifyField(String path, int value) {
        Assert.assertEquals((int) response.jsonPath().getInt(path), value);
    }
}

SECTION 6

Parallel Execution Deep Dive — Multithreading in Frameworks

The most asked senior SDET interview topic. Understand how parallel execution works under the hood.

Why Parallel Execution?
A test suite of 500 tests taking 3 seconds each = 25 minutes sequential. With 5 parallel threads = ~5 minutes. In CI/CD, fast feedback is critical. Parallel execution is not optional — it's expected.

🔧 Java + Selenium + TestNG: ThreadLocal Pattern

! The Problem
Selenium WebDriver is NOT thread-safe. If you use a static WebDriver reference and run tests in parallel, all threads share the same driver instance → race conditions → random failures.

Solution: Java's ThreadLocal<WebDriver> class. It creates a separate copy of the variable for each thread. Thread A's WebDriver is invisible to Thread B.
⚙ ThreadLocal WebDriver Factory
public class DriverManager {
    private static ThreadLocal<WebDriver> driver = new ThreadLocal<>();

    public static void setDriver(WebDriver d) {
        driver.set(d);
    }

    public static WebDriver getDriver() {
        return driver.get();
    }

    public static void removeDriver() {
        driver.get().quit();
        driver.remove();  // CRITICAL: prevents memory leaks
    }
}
⚙ TestNG XML Configuration
<!-- 4 parallel modes available -->

<!-- 1. Parallel by METHODS — each @Test method in its own thread -->
<suite parallel="methods" thread-count="5">

<!-- 2. Parallel by CLASSES — each class in its own thread -->
<suite parallel="classes" thread-count="5">

<!-- 3. Parallel by TESTS — each <test> tag in its own thread -->
<suite parallel="tests" thread-count="5">

<!-- 4. Parallel by INSTANCES — same instance methods in same thread -->
<suite parallel="instances" thread-count="5">
Most common for Selenium: parallel="tests" where each <test> tag targets a different browser. Combined with ThreadLocal, each browser gets its own thread.
⚙ DataProvider Parallel Execution
@DataProvider(name = "browsers", parallel = true)
public Object[][] browserData() {
    return new Object[][] {
        { "chrome" }, { "firefox" }, { "edge" }

@Test(dataProvider = "browsers")
public void crossBrowserTest(String browser) {

    driver.get("https://example.com");

🎭 Playwright: Parallel by Default

⚙ How Playwright Parallelism Works
Playwright uses a worker-based model:

• Each test file (spec file) runs in a separate worker process
• Each test within a file gets its own isolated BrowserContext (separate cookies, session, storage)
• No shared state between tests = no race conditions
• No ThreadLocal needed — isolation is built-in

Configure in playwright.config.ts:
{
  workers: 4,               // 4 parallel worker processes
  fullyParallel: true,       // parallelize tests WITHIN files too
  retries: 1,                // auto-retry failed tests once
}

// Run with CLI override:
npx playwright test --workers=8
⚙ Playwright Sharding (CI/CD Scale)
For massive test suites across multiple CI machines:
# Machine 1:
npx playwright test --shard=1/3

# Machine 2:
npx playwright test --shard=2/3

# Machine 3:
npx playwright test --shard=3/3

# Split 300 tests across 3 CI nodes = 100 each, running in parallel

📋 Parallel Execution Comparison Cheat Sheet

Selenium + TestNG: Configure in testng.xml → parallel="methods|classes|tests" + thread-count → Use ThreadLocal<WebDriver> for thread safety → driver.remove() in teardown

Playwright: Set workers in config → Tests isolated via BrowserContext automatically → Use fullyParallel: true for within-file parallelism → Shard across CI machines with --shard

Cucumber + TestNG: Override scenarios() with @DataProvider(parallel=true) → Set data-provider-thread-count in testng.xml → Each scenario runs in its own thread

Selenium Grid: Hub distributes tests to Node machines → Each node runs a browser instance → Combine with TestNG parallel for max throughput → Grid 4 uses Docker for easy node scaling
💡 Interview Golden Answer: "In my framework, I use ThreadLocal<WebDriver> with a DriverFactory class following the Singleton pattern. Each thread creates its own WebDriver instance via ThreadLocal.set(), and in @AfterMethod, I call ThreadLocal.remove() to prevent memory leaks. In testng.xml, I configure parallel='methods' with thread-count=5. For API tests, I use Cucumber's DataProvider with parallel=true since they're stateless and naturally thread-safe."
